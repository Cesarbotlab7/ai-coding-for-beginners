## 从看到做，先别急着装一堆工具

前面几章，你已经搞清楚了一件事：Claude Code 是一个跑在终端里的 AI 助手，它能读写你电脑上的文件、帮你写代码、修 bug。

但那都是"知道"。

"知道"和"用上"之间，不只隔着一条安装命令，还隔着一套能让你少折腾的基础装备。

我在短视频里看到一套 vibe coding 入门建议，里面有一个判断是对的：新手最大的坑，往往不是模型不够强，而是电脑、编辑器、网络、输入方式这些基础设施没配好。你还没开始做产品，就先被环境问题耗光耐心。

这一章先把装备选明白，再走 Claude Code 的安装步骤。

---

## 先配一套能开工的装备

vibe coding 不是"会背命令"，而是你能把一个想法连续说清楚，让 AI 在项目里一步步改出来。

所以工具选择的顺序应该是：

**先少折腾**：电脑和网络要稳定，不要天天卡在安装报错。

**再看得见**：编辑器要能看到文件、改动、预览结果。

**最后才追求上限**：Claude Code、Codex 这类 Agent 工具很强，但更适合在你已经能理解"它准备改哪些文件"之后使用。

## 这套教程默认走哪条路线

这套教程不做工具横评，也不追当天最热的模型。

默认路线是：

**Mac 或一台稳定电脑 → VS Code → Claude Code → GitHub → 本地跑通 → 再部署上线。**

如果你已经在用 Cursor、Windsurf、Trae 或 Codex，也可以继续用。你需要抓住的是同一套工作方式：

- 让 AI 先读项目，不要凭空写
- 大改动先让 AI 列计划，不要直接开干
- 改完让 AI 跑测试或构建，不要只看它说"完成了"
- 价格、套餐、模型名字变化很快，付费前看官网，不要照抄别人截图里的数字

## 接下来需要准备什么

四样东西：

**一台 Mac 或 Windows 电脑**——Linux 也行，但这个教程以 Mac 为主。

**一个可用的 AI 编程账号**——本章主线用 Claude Code。你可以用 Claude.ai 账号登录，也可以使用 Anthropic API。不同套餐、额度和计费方式会变，实际以官网为准。

**5 分钟**——认真的，就这么快。

**稳定的网络环境**——安装工具、登录账号、下载依赖都需要网络。网络不稳定时，不要先怀疑自己不会编程，很多时候只是工具连不上。

## 什么是「终端」

安装过程需要用到「终端」。终端不是什么神秘工具，它就是一个让你**用文字命令控制电脑**的窗口。

你平时用鼠标点击来做事，终端是用打字来做同样的事。

**Mac 打开终端**：按 ⌘ + 空格，输入「终端」，回车。或者在 Launchpad 里找「终端」。

看到一个黑色或白色的窗口，里面有个光标在闪，就对了。

## 装好之后，能做什么

Claude Code 启动之后，你可以直接用中文跟它说话：

> 帮我创建一个 HTML 页面，显示"Hello World"，字体大一点，居中

它会在当前文件夹里创建文件，你用浏览器打开就能看到。

这就是你的第一个「用 AI 写的程序」。
