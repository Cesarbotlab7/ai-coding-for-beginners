## 你不会写代码

这没问题。

这个教程里的 AI 资讯平台，80% 以上的代码是 Claude Code 写的。前端、爬虫、测试、CI 配置——绝大多数代码，作者没有手写，而是用自然语言描述需求，Claude Code 负责实现。

你不需要先学会写代码，才能开始做产品。

你需要的，是学会**描述你要的东西**。

---

## Claude Code 是什么

Claude Code 是 Anthropic 的命令行 AI 编程助手。你在终端里叫它，它会：

- 读你的项目文件，理解上下文
- 根据你的需求写代码、改代码、找 bug
- 在本地直接改文件，你看到的就是改完的版本
- 跑测试、跑构建，确认改动是否有效

和 ChatGPT 不同的地方：它不是给你一段代码让你自己粘贴——它直接在你的文件系统里操作。

---

## 两种工作模式

Claude Code 有两种主要的工作方式，你需要知道什么时候用哪种：

**直接描述（Action 模式）**

适合：改一个小地方、加一个简单功能、修一个已知的 bug。

你说："把首页的字体大小从 14px 改成 16px"，它改完了。

**Plan 模式（先列计划）**

适合：做一个新功能、重构一块代码、不确定影响范围的改动。

你在描述前加上"先用 Plan 模式"，它会先梳理影响的文件、列出实现步骤、提出需要你决策的问题，等你确认后再动手。

## 怎么描述一个好需求

描述需求的核心是三件事：

**目标是什么**

不是"改一下"，而是"我想让用户能按 Twitter/RSS/HN/arXiv 分类筛选资讯，点 Tab 切换"。

**约束是什么**

"不要动数据库的结构"、"不要加新的 npm 包"、"用现有的 CSS 变量，不要硬编码颜色"。

**不确定的部分**

"我不知道这个改动会不会影响搜索功能，你来判断一下"。

三件事说清楚了，Claude Code 给出的结果往往第一次就对。

---

## 什么时候该说"停"

AI 助理很能干，但不是万能的。几个需要你主动把握方向的场景：

- **需求本身没想清楚**：AI 会按你说的做，但你说的不是你真正要的
- **改动范围超出预期**：让它先 Plan，看清楚会碰哪些文件再决定
- **发现错误方向**：早说"停，重来"比等它改完再推倒重来便宜
- **涉及安全/密钥**：永远不要把 API key 粘进对话，让 AI 从环境变量读
