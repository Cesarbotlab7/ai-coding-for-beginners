## 你去一家餐厅

你走进一家餐厅。

你看到的第一眼是**门面**——招牌、灯光、摆设、菜单、服务员的脸。这些东西决定你愿不愿意进来，愿不愿意待下去。

你看不到的，是**后厨**——厨师、炉火、原料、配方。你点单之后那道菜是怎么出来的，你不知道，也不需要知道。

后厨后面还有**仓库**——原料从这里来，账本在这里记。

整个餐厅坐落在一个**地址**上——没有地址，就算菜再好，别人也找不来。

店外立着一块**招牌**——路人看到招牌，才知道往这里走。

这五样东西，互联网产品里都有，只是换了名字：

| 餐厅 | 互联网产品 |
|------|------------|
| 门面 + 菜单 + 服务员 | **前端** |
| 后厨 | **后端** |
| 仓库 | **数据库** |
| 店铺地址 | **服务器** |
| 店外招牌 | **域名** |

## 服务员 vs 厨师

餐厅里最容易搞混的两个角色，是**服务员**和**厨师**——也就是前端和后端。

服务员（前端）负责什么？

- 把菜单展示给你看
- 把你点的单子记下来
- 把菜端上桌、摆好
- 微笑、解释、处理投诉

厨师（后端）负责什么？

- 接收点单，搞清楚要做什么
- 炒菜、控火候、计算分量
- 管库存、知道仓库里有什么
- 不面对客人，但菜是从这里出来的

**关键点：服务员不会炒菜，厨师不会点单。** 两件事天然要分开。

同一个"点赞"按钮，前端和后端各干什么？

- **前端**：按钮长什么样、点了之后是否有动效、数字显示在哪里
- **后端**：这次点赞记没记进数据库、这个用户有没有重复点赞、总数是多少

## 前端三件宝

前端有三样基础工具，你会慢慢在各处听到它们的名字：

| 工具 | 类比 | 干的事 |
|------|------|--------|
| HTML | 骨架 | 决定"有什么"（标题、图片、按钮） |
| CSS | 衣服 | 决定"长什么样"（颜色、大小、布局） |
| JavaScript | 肌肉 | 决定"能动吗"（点击、动画、交互） |

你现在不需要学会写这三样东西。但你需要知道，下次有人说"这是个 JS 问题"或者"CSS 写一下就好"，他说的是什么范围的事。

## 在真实项目里

AI 资讯平台的餐厅长这样：

- **前端**（`web/` 目录，Next.js）：你看到的资讯卡片列表、信源 Tab 筛选（Twitter / RSS / HN / arXiv）、移动端适配、阅读时长标签
- **后端**（`scraper/` 目录，Python）：每 3 小时从四个渠道抓取资讯、去重、按重要程度打分、用 AI 生成中文摘要、写进 Supabase 数据库
- **数据库**（Supabase）：存着几千条资讯记录，每次前端请求都从这里查
- **服务器**（Vercel）：前端部署在这里，24 小时开着，全球可访问
- **域名**：用来替代那串让人记不住的 IP 地址

它们合在一起，才是你在浏览器里看到的那个页面。
