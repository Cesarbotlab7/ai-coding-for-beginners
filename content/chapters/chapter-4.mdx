## 你输入了账号密码

你打开一个 App，输入账号和密码，点"登录"。

不到一秒，登录成功了。

App 是怎么知道"这个密码是这个账号的"的？

答案是：它翻了一个本子。那个本子，就是数据库。

---

## 数据库是什么

数据库是一个**被程序自动读写的 Excel 表格**。

听起来很普通，但这个比喻很准：

| Excel | 数据库 |
|-------|--------|
| 一张工作表 | 一张"表"（table） |
| 一行记录 | 一行数据（row） |
| 一列字段 | 一列属性（column） |
| 筛选、排序 | 查询（query） |
| 手动打开修改 | 代码自动读写 |

区别在最后一行：Excel 是人打开来手动改的。数据库是程序在后台自动存取的，一秒钟可以操作几千条记录，完全不需要人盯着。

---

## 为什么不直接用 Excel

用过 Excel 共享编辑功能的人都知道，多个人同时改一个文件，很容易乱。

数据库解决了这个问题：它专门设计来处理"同时有几百个人在读写"的场景。而且数据量大的时候，数据库查询比 Excel 滤波快几个数量级。

## AI 资讯平台的数据库长什么样

这个平台用的是 Supabase，一个云端数据库服务。核心是一张叫 `news_items` 的表：

| 字段名 | 存什么 |
|--------|--------|
| `url_hash` | 每条资讯的唯一 ID（用 URL 算出来的） |
| `title` | 标题 |
| `source_type` | 来源（twitter / rss / hackernews / arxiv） |
| `source_tier` | 来源重要级别（1 = 顶级人物，4 = 社区） |
| `importance_score` | 重要性分数（0-100，越高越靠前） |
| `ai_summary_zh` | AI 生成的中文摘要（英文内容才有） |
| `is_representative` | 是否在首页显示 |
| `fetched_at` | 抓取时间 |

你在首页看到的每一张资讯卡片，都是从这张表里查出来的：先过滤 `is_representative=true`，再按 `importance_score` 从高到低排列。

就这一句查询，决定了你看到什么、先看到什么。

## 数据库不需要你会写

理解数据库，不是要你会写 SQL 查询语句。

你只需要知道：当你问"这个功能的数据存在哪？"——答案通常是某张表的某几列。知道这一点，你就能跟程序员或者 AI 说清楚你的需求。
