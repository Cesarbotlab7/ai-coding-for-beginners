# DESIGN.md — WorkBuddy 教程设计系统

本文档定义网站唯一视觉与交互语言。具体值统一写入 `src/styles/tokens.css`，组件只引用 token。

## 一、设计目标

1. **像一张清楚的操作台**：读者始终知道现在在哪、下一步做什么、完成的信号是什么。
2. **安静但不单调**：正文保持阅读感，交互区域明显可识别。
3. **安全感优先**：高风险、待确认、已完成必须有文字和图标，不只依赖颜色。
4. **模拟而非仿冒**：WorkBuddy 工作台使用自制简化界面，不逐像素复制官方产品。
5. **小屏可操作**：桌面三栏在移动端改为纵向步骤，不做横向缩小。

## 二、视觉基调

- 主背景：暖白或深灰。
- 卡片：纯色、细边框、无装饰阴影。
- 主强调：信息蓝。
- 成功、警告、危险只表达状态。
- 不使用渐变、霓虹、玻璃模糊、粒子和动画边框。
- 不使用 emoji 作为 UI 图标；图标使用内联 Lucide 风格 SVG。

## 三、Token 类别

`tokens.css` 必须包含：

- 背景、文字、边框和语义颜色。
- sans、serif、mono 字体。
- 字号、行高、字重。
- 4px/8px 基础间距。
- 圆角和边框宽度。
- 正文、首页、工作台最大宽度。
- 控件最小高度、焦点环和快速过渡。

组件中禁止出现：

- 十六进制、rgb、hsl 等硬编码颜色。
- 任意像素间距、字号、圆角或宽度。
- `box-shadow` 装饰。
- 超过 token 定义的粗边框。

百分比、`fr`、`auto`、`minmax()` 和内容相关布局值可以直接使用。

## 四、排版

- 正文宽度：`--content-width`。
- 首页宽度：`--home-width`。
- 工作台宽度：`--workbench-width`。
- 正文使用 sans，行高宽松。
- 引语和少量复盘句可使用 serif。
- 路径、文件名、模式名和模拟日志使用 mono。
- 标题只用 regular 和 medium 两种字重。

正文层级：

- 页面标题：`--text-2xl`。
- 章节标题：`--text-xl`。
- 二级标题：`--text-lg`。
- 小标题：`--text-md`。
- 正文：`--text-base`。
- 说明和元信息：`--text-sm` / `--text-xs`。

## 五、基础组件

### 按钮

- 最小高度使用 `--control-height`。
- 默认透明背景、细边框。
- 主按钮使用正文色背景和页面背景色文字。
- 危险操作使用危险语义色并保留明确动词。
- `:hover`、`:focus-visible`、`:active` 和 disabled 状态必须齐全。

### 卡片

- 主卡片：主背景、细边框、`--radius-lg`。
- 次级区块：次级背景。
- 选中状态：信息背景 + 文本标签或勾选图标。

### 状态提示

统一四类：

- 信息
- 成功
- 警告
- 危险

每类同时包含标题、说明和图标/文字标签。

### 表单

- label 永远可见。
- 输入框、textarea、select 使用统一边框和焦点环。
- 错误信息紧跟字段，并通过 `aria-describedby` 关联。
- 不能用 placeholder 代替 label。

## 六、WorkBuddy 模拟工作台

桌面布局：

1. 左栏：任务与功能入口。
2. 中栏：任务描述、计划和执行过程。
3. 右栏：产物、预览和验收。

工作台必须：

- 使用 `--workbench-width`，允许突破正文宽度。
- 在小屏改为“任务 → 执行 → 结果”的纵向顺序。
- 每个区域有可见标题。
- 当前区域既有背景变化，也有 `当前` 文本。
- 模式、权限、工作空间始终显示为独立控件。

## 七、动效

- 只用于状态切换、进度和展开收起。
- 持续时间使用 `--transition-fast` 或 `--transition-base`。
- 不做循环动画。
- `prefers-reduced-motion: reduce` 时取消平滑滚动、位移和非必要过渡。

## 八、响应式

移动端断点由 `--breakpoint-mobile` 记录在文档，但媒体查询仍使用对应固定值，因为 CSS 自定义属性不能直接用于媒体查询。

移动端：

- 页面两侧保留 `--page-gutter-mobile`。
- 导航允许换行。
- 多栏卡片变单栏。
- 工作台变步骤式纵向布局。
- 点击目标不小于 `--control-height`。

## 九、可访问性

- 对比度达到 WCAG AA。
- `:focus-visible` 使用统一焦点环。
- 图标按钮有可访问名称。
- 装饰 SVG 使用 `aria-hidden="true"`。
- 交互结果写入 `aria-live="polite"` 区域。
- 选项组优先使用原生 radio/checkbox。
- 页面提供跳到正文链接。

## 十、设计验收

- 搜索组件样式，不出现硬编码颜色。
- 搜索组件样式，不出现未解释的 px 值。
- 360px、桌面、浅色、深色、reduced-motion 均完成检查。
- 同一组件渲染两次不产生重复 id 或跨组件状态污染。
