# SPEC.md — 功能规格

本文档定义教程网站的每一个页面、每一章内容、每一个交互组件的具体规格。所有实现必须对照本文档进行。

---

## 网站整体结构

### 路由

| 路径 | 说明 | 源文件 |
|------|------|--------|
| `/` | 首页：整个系列的目录与介绍 | `src/pages/index.astro` |
| `/about` | 关于作者、关于这个系列 | `src/pages/about.astro` |
| `/chapter/[slug]` | 章节详情页（动态路由） | `src/pages/chapter/[slug].astro` |

### 首页需要展示的内容

1. **顶部 Hero 区**：系列名 + slogan + 一句引导
2. **整个系列的"阶段-章节"大纲**：4 个阶段（引子 / 地图 / 玩家 / 动手），每个阶段下列出对应章节卡片
3. **每张章节卡片**：编号、标题、一句话摘要、状态（已发布 / 准备中）、阅读时长估算
4. **关于作者**：简短自我介绍 + 联系方式

### 章节详情页布局

```
┌─────────────────────────────────────┐
│ 顶部导航（回首页 · 上一章 · 下一章）│
├─────────────────────────────────────┤
│                                     │
│  阶段标签 · 第N章                   │
│  ═══════════════════════════════    │
│  章节标题                           │
│  一句话摘要                         │
│  更新日期 · 预计阅读时长            │
│                                     │
│  [正文 + 交互组件混合排版]          │
│                                     │
├─────────────────────────────────────┤
│  上一章  |  下一章                  │
└─────────────────────────────────────┘
```

正文区宽度约 680px（和我的聊天里的 widget 容器等宽，设计决策统一）。

### 内容流

**1. 开场（前导段落 + `ImaginationVsReality` 组件）**

前导段落（Markdown 正文）：两段文字，讲"小白打开一个程序员的电脑会看到什么"。第一段描述一种普遍的幻想（黑客电影式的滚动代码），第二段引出真相："真实项目其实只是几个文件夹"。

然后渲染 `ImaginationVsReality` 组件（见下方组件规格）。

**2. 过渡段**

几句话引导读者："接下来我会把这个真实项目的目录打开给你看。你不需要记住任何技术名词，只需要对每一个成员有个印象——就像看球赛前先认识每支队伍。"

**3. 主体（`FileExplorer` 组件）**

`FileExplorer` 组件渲染。这是整章的重头戏，占据屏幕大部分空间。

**4. 过渡段**

一段文字过渡："你刚才点完了几个文件夹。先别怀疑自己是否真的学到了什么——继续往下看。"

**5. 收尾（`ChapterCloser` 组件）**

列出"四件事"，一句引语，以及下一章预告。

### `FileExplorer` 组件规格

**作用**：模拟 Windows 资源管理器，让读者能真实地在网页里"打开"AI热点资讯平台这个项目的文件夹，点击每个文件/文件夹能看到说明。

**核心数据结构**：

```ts
type FileNode = {
  name: string;          // 显示名，如 "scraper" 或 "CLAUDE.md"
  type: 'folder' | 'file';
  size?: string;         // 文件大小，如 "7 KB"；文件夹为空
  key: string;           // 唯一标识，用于追踪"已看过"
  role: string;          // 短标签，如 "后端"、"配置"、"说明书"
  nick: string;          // 中文昵称，如 "信鸽"
  desc: string;          // 详细描述（1-3 句）
  hint?: string;         // 蓝框小白认知金句
  enterable?: boolean;   // 是否可以"点进去"（仅 scraper 和 web 为 true）
};

type Folder = {
  name: string;
  path: string[];
  items: FileNode[];
};
```

**顶层目录、scraper 子目录、web 子目录**的完整节点数据，见 `content/data/file-explorer-nodes.ts`（此文件作为独立数据源，便于后期修改）。

**UI 要点**：

- 顶部面包屑：`AI热点资讯平台 > scraper`
- 顶部有"返回上一级"按钮，在根目录时禁用
- 列表按三列展示：名称 / 类型 / 大小
- 点击列表项：右侧（或下方）详情区展示该节点的昵称、角色标签、描述、hint
- 文件夹图标（仅 scraper、web 可进入）右侧详情区底部出现"点进去看看 →"按钮
- 底部进度计数："已认识 N / 11+ 个成员"

**状态与行为**：

- 当前视图文件夹 `currentFolder`（默认 `root`）
- 已看过的节点集合 `seenKeys`（用于进度计数，不持久化）
- 进入子文件夹时清空详情区
- 键盘支持：上下键切换选中项，Enter 打开文件夹，Backspace 返回上级

**样式要点**：

- 图标：文件夹用 Windows 风格的"黄色文件夹"svg，文件用灰色"带折角"svg
- 行高 40px，hover 变灰，选中变蓝（`--color-bg-info`）
- 详情区用 `--color-bg-secondary` 背景
- "小白认知金句"用 `--color-bg-info` 背景块强调

**交互脚本**：原生 JS 即可，避免引入框架。

**数据完整性**：所有节点的 `desc` 和 `hint` 必须严格遵守"用比喻替代术语"的核心原则。昵称列表（仅供核对）：

顶层：

- `scraper` → 出门收信的信鸽

scraper 子目录：

web 子目录：

### `ChapterCloser` 组件规格

**作用**：每章末尾的通用收尾组件，展示"本章学到的要点 + 一句引语 + 下一章预告"。

**Props**：

```ts
type ChapterCloserProps = {
  takeaways: string[];        // 3-5 条，每条是一句话
  quote: { text: string; attribution: string };
  next?: {                    // 下一章信息；最后一章为 null
    chapterNumber: number;
    title: string;
    description: string;
  };
};
```

**第0章实际内容**：

```ts
takeaways: [
  "一个真实项目，也就十几个文件。你一眼看完这个文件夹的能力，已经超过了 80% 完全不懂编程的人。",
  "项目里有两种东西：你写的 + 工具自动生成的。点开头的文件夹（.claude、.github、.next）绝大部分都是工具管的，你不用怕。",
  "一个完整产品，通常是\"后端 + 前端\"的双项目结构。scraper 负责收集数据、web 负责展示给用户——这两件事天然要分开。",
  "给 AI 写的说明书（CLAUDE.md）是新时代的必需品。你会越来越多地看到这种\"写给 AI 看\"的文件——这是过去三年才出现的新物种。"
],
quote: {
  text: "编程的可怕之处不在于它复杂，而在于第一次看到它的时候没人告诉你：它其实就是几个文件夹装着几个文件而已。",
  attribution: "每一个走过这条路的人都会同意"
},
next: {
  chapterNumber: 1,
  title: "一个产品，是怎么从你脑子里，跑到别人手机里的？",
  description: "你已经认识了每个文件夹是谁。下一章我们按下启动键——看用户点开你这个网站的那一秒，这些文件夹之间到底发生了什么。"
}
```

**UI 要点**：

- "其实你已经学到了 四件事"作为标题（"四件事"动态根据 takeaways 长度变化）
- 每条 takeaway 前面带一个绿色对勾圆
- 引语用 serif 字体，左侧带 2px 竖线
- 下一章预告用 `--color-bg-info` 卡片，含"开始第 N 章 →"按钮

## 测试要求

项目上线前（Phase 3 之后），每章必须通过以下检查：

1. **移动端适配**：在 360px 宽度下无横向滚动条、所有文字可读、所有交互可点
2. **深色模式**：切换系统深色模式后无"白底黑字"漏网之鱼
3. **键盘可达**：只用 Tab 键能到达每一个交互元素
4. **Lighthouse**：性能 / 可访问性 / SEO 各项不低于 90 分
